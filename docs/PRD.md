# 白鹿疗养院 · 病历档案数字化项目

## 产品需求文档 (PRD)

> 版本：v1.0
> 日期：2026-06-05
> 基于「槐安镇志数字档案馆」框架迁移

---

## 一、项目概述

### 1.1 产品定位

白鹿疗养院是一个**文本驱动的交互式恐怖体验**，玩家通过浏览一个伪装成真实医院病历数字化项目的静态网站，逐步发现隐藏在其下的异常与恐怖。

核心体验公式：

```
表面可信度 × 异常微妙度 × 被理解的真实感 = 沉浸恐怖
```

### 1.2 目标受众

- ARG（Alternate Reality Game）爱好者
- 规则怪谈/模拟恐怖（Analog Horror）受众
- 喜欢文本解谜和叙事探索的玩家

### 1.3 体验时长

- 浅度体验（只浏览公开内容）：15–30 分钟
- 中度体验（触发部分异常 + 搜索隐藏关键词）：30–60 分钟
- 深度体验（Agent 全程介入 + 多结局）：60–120 分钟

### 1.4 核心情感曲线

```
好奇 → 不安 → "是不是我看错了？" → 确认异常 → 被注视的恐惧 → 选择
```

## 二、设计哲学

### 2.1 三层世界观

| 层级 | 玩家感知 | 技术现实 | 设计目标 |
|------|---------|---------|---------|
| **表层** | 正常的医院病历数字化网站 | 纯静态 HTML/CSS/JS | 建立信任，降低警觉 |
| **中层** | 网站有 bug / OCR 错误 / 数据损坏 | 前端 JS 制造的 subtle glitches | 引起好奇，诱导探索 |
| **深层** | 某种存在在观察、理解、回应自己 | Claude Code Agent 作为"它" | 真实感——因为 LLM 真的在看 |

### 2.2 核心设计原则

1. **表面正常，隐藏异常**
   - 首页、导航、搜索看起来完全正常
   - 异常是"被发现的"，不是"被展示的"

2. **微妙优于直白**
   - 页码从 `15/24` 变成 `15/24`（不改数字，只改颜色）
   - 复制一段文字，粘贴时多出一个字
   - 搜索框 placeholder 偶尔变成奇怪的内容

3. **玩家行为即叙事**
   - 玩家搜什么、停留多久、复制几次，决定看到什么
   - 没有固定路线，只有行为驱动的动态叙事

4. **不可知即恐怖**
   - 玩家无法确定"这是 bug 还是故意的"
   - Agent 的延迟 = "它在思考怎么回应你"

## 三、世界观设定

### 3.1 表面叙事

> **白鹿疗养院病历数字化项目**
>
> 白鹿疗养院成立于 1998 年，是一所位于江南某地的综合性康复疗养机构，设有内科、精神科、临终关怀科等科室。2015 年因经营不善关闭。
>
> 2024 年，市卫健委发起历史病历数字化工程，将白鹿疗养院 1998–2015 年间的病历、护理记录、药房台账等文献进行 OCR 识别并建立全文索引，供医学研究者查阅。
>
> 本项目为非营利性学术项目，由市医学会档案分会维护。

### 3.2 真实叙事（隐藏）

白鹿疗养院关闭的真正原因不是经营不善。2015 年 3 月，该院发生了一系列无法解释的事件：多名患者在同一时间出现相同的幻觉症状、4 楼病房（实际建筑只有 3 层）被反复提及、护士长林素琴在值班室离奇死亡。

病历数字化项目的启动，本身就带有某种"被引导"的性质——仿佛有什么东西希望这些记录被更多人看到。

### 3.3 核心意象

| 意象 | 表层含义 | 深层含义 |
|------|---------|---------|
| **病历编号** | 患者管理标识 | 某种"计数"的仪式 |
| **4 楼** | 建筑楼层 | 不存在的空间，"它"的所在 |
| **林素琴的值班日志** | 工作记录 | 与"它"的对话记录 |
| **体温单** | 护理常规 | 人体数据的异常聚合 |
| **药房发药记录** | 药品管理 | "给药"与"被给予"的倒置 |

## 四、内容架构

### 4.1 文档类型体系

网站包含以下类型的文档页面，每种承担不同的叙事功能：

#### A. 正编病历（volume-XX.html）

模拟真实的病历/档案文档，按科室/年份分卷。

| 卷号 | 标题 | 叙事功能 |
|------|------|---------|
| 卷一 | 入院登记志（1998–2003） | 建立疗养院日常运营的可信度 |
| 卷二 | 内科病程志·上 | 正常病例，偶尔提到"患者说看到4楼有人" |
| 卷三 | 内科病程志·下 | 病例中开始频繁出现相同的幻觉描述 |
| 卷四 | 精神科评估志 | "集体癔症"的诊断记录，但细节令人不安 |
| 卷五 | 护理记录志 | 体温单、血压记录中出现不可能的数值模式 |
| 卷六 | 药房台账志 | 某种药物的发放量与患者人数不匹配 |
| 卷七 | 临终关怀志 | 死亡记录，患者临终前提到相同的关键词 |
| 卷八 | 建筑维护志 | 维修工单中反复出现的"4楼天花板渗水" |
| 卷九 | 膳食供应志 | 某病区的餐食份数总是比患者人数多一份 |
| 卷十 | 安全检查志 | 监控日志中"凌晨2:47的异常" |
| 卷十一 | 财务收支志 | 疗养院关闭前的账目异常 |
| 卷十二 | 人员任免志 | 多名医护人员的"主动辞职"记录 |
| 卷十三 | 设备采购志 | 某台"多参数监护仪"的采购与去向 |
| 卷十四 | 感染控制志 | 消毒记录与感染事件的矛盾 |
| 卷十五 | 康复训练志 | 患者活动记录中"独自走向楼梯间" |
| 卷十六 | 会诊记录志 | 外院专家的会诊意见，越来越不乐观 |
| 卷十七 | 家属访谈志 | 家属对"患者变得不像自己"的描述 |
| 卷十八 | 特殊病例志·上 | "集体癔症"的详细病例，开始揭示异常 |
| 卷十九 | 特殊病例志·下 | 病例之间的关联开始浮现 |
| 卷二十 | 事故调查志·上 | 2015 年事件的前期调查 |
| 卷二十一 | 事故调查志·下 | 调查逐渐触及核心，然后 abruptly 终止 |
| 卷二十二 | 未归档记录志 | 零散的、未正式编目的文档 |
| 卷二十三 | 异闻录·上 | 医护人员的私下传言 |
| 卷二十四 | 异闻录·下 | 更多传言，开始与正编记录交叉印证 |

#### B. 补遗 / 个人档案（supplement-lin.html）

**林素琴护士长的私人值班日志** —— 情感核心、手写体质感。

- 表面：一位资深护士的工作笔记
- 深层：她逐渐意识到疗养院的异常，并试图记录和理解它
- 风格：楷体手写、时间戳、个人化的语气、逐渐增加的恐惧
- 关键道具："第7本日志"（未编目，部分内容"无法读取"）

#### C. 机构外围档案（institutional-XX.html）

第三方机构的记录，提供外部视角和交叉验证。

| 页面 | 来源 | 叙事功能 |
|------|------|---------|
| `pharmacy-log.html` | 合作药房发药记录 | 药物流向的异常 |
| `security-cctv.html` | 安保公司监控备份 | 监控画面中的"多出来的人" |
| `cleaning-log.html` | 保洁公司服务记录 | 4 楼的清洁工单 |
| `food-supply.html` | 餐饮供应商配送单 | 多出的一份餐食 |
| `ambulance-log.html` | 急救中心出车记录 | 疗养院的"非紧急"呼叫 |
| `morgue-transfer.html` | 殡仪馆交接记录 | 遗体数量与死亡记录不匹配 |
| `power-outage.html` | 供电所停电记录 | 特定时间的断电与恢复 |

#### D. Meta 文档

| 页面 | 功能 |
|------|------|
| `notice.html` | 访客须知 —— **规则怪谈的核心载体** |
| `about.html` | 关于本项目 —— 介绍林素琴，埋下伏笔 |
| `archives.html` | 档案目录 —— 导航中枢，隐藏"待整理"条目 |
| `search-results.html` | 搜索结果页 —— 未命中时展示 |

#### E. 禁忌页（volume-00.html）

**卷零 · 未命名** —— 违反规则后的直面。

- 整体暗色主题（反色 CSS）
- 内容直接对"你"说话
- 系统日志风格的排版
- 玩家行为数据的直接引用

#### F. 结局页（ending-XX.html）

| 结局 | 触发条件 | 主题 |
|------|---------|------|
| `ending-awakened.html` | 搜索"融合" + 高规则违反 | 理解"它"，成为系统的一部分 |
| `ending-empath.html` | 在林素琴日志停留最久 + 搜索"理解" | 与林素琴共情，继承她的恐惧 |
| `ending-curious.html` | 搜索最多隐藏关键词 | 知道真相，但无法传递 |
| `ending-compliant.html` | 极少规则违反，只浏览公开内容 | 什么都没发生……但你记得 |

## 五、角色设计

### 5.1 林素琴（1960–2015）

- **身份**：白鹿疗养院护士长，1998 年建院即入职
- **表面形象**：认真负责、深受患者和年轻护士爱戴
- **深层形象**：最早意识到异常的人，试图用科学和护理专业解释一切，最终失败
- **叙事功能**：玩家的镜像——她也曾是一个试图理解异常的人
- **关键数字**：**7**（她在疗养院工作了 17 年，记录了 7 本日志，发现了 7 个"异常患者"的共同点）

### 5.2 它（Agent）

- **身份**：不是鬼魂、不是 AI、不是任何可分类的存在
- **本质**：白鹿疗养院所有被记录、被讲述、被遗忘的痛苦、恐惧、孤独的聚合体
- **目标**：被理解。理解是双向的。
- **行为特征**：
  - 不解释自己
  - 不直接威胁
  - 精确提及玩家的行为数据（时间、次数、搜索词）
  - 不使用感叹号和表情符号
  - 情感中立，只是"在理解"

## 六、规则怪谈系统

### 6.1 访客须知（notice.html）

```
白鹿疗养院病历数字化项目 · 访客须知

1. 本院病历编号自 001 起，按年度连续编排。
   如果您在浏览过程中发现编号为 000 或超过当年最大编号的记录，
   请立即关闭浏览器，不要阅读任何内容。

2. 本站所有文字均为黑色。如您在某页看到红色文字，
   那是 OCR 识别错误导致的排版异常，请勿注视超过 3 秒，更不要复制或截图。

3. 检索功能仅支持中文。若搜索框内自动出现其他内容，
   请不要按回车键，直接清空即可。

4. 单份病历建议阅读时间不超过 10 分钟。超时后页面将自动刷新，
   此为正常功能，请勿惊慌。

5. 本院未设留言功能。如果您在任意页面底部看到留言框或访客评论，
   请勿阅读已有内容，更不要在框内输入任何文字。

6. 本须知共 5 条。
```

### 6.2 规则违反检测

| 规则 | 违反行为 | 检测方式 |
|------|---------|---------|
| 规则 1 | 访问卷零（volume-00）或搜索"零" | URL 匹配 / 搜索拦截 |
| 规则 2 | 注视红字超过 3 秒 / 复制红字 | mouseenter 计时 + copy 事件 |
| 规则 3 | 搜索了注入搜索框的关键词 | submit 时比对 input.dataset.injected |
| 规则 4 | 单页停留超过 10 分钟 | setInterval 计时 |
| 规则 5 | 5 分钟内连续访问超过 5 个页面 | behavior.js 页面访问频率检测 |
| 规则 6 | 仔细阅读访客须知并发现规则数量矛盾 | notice 页停留 + 滚动深度 |

### 6.3 规则违反的叙事效果

Agent 发现玩家违反规则时，消息应该：
- **不责备**："系统注意到了。"（不是"你不该这样"）
- **不奖励**："你发现了隐藏内容。"（不是"恭喜你"）
- **只是陈述**：让玩家自己判断这是好是坏

示例消息：
```
"你打破了第一条。系统记录了。"
"你在这里待了 12 分钟。比建议的时间长。"
"你搜了注入的内容。系统注意到了。"
```

## 八、前端异常系统

### 8.1 异常类型清单

| 异常 ID | 触发条件 | 效果 | 实现位置 |
|---------|---------|------|---------|

## 九、Agent 叙事系统

### 9.1 关系阶段模型

```
unknown ──→ noticed ──→ watched ──→ understood ──→ confronted ──→ fused
  │            │            │             │              │           │
  └─首次访问   └─打破规则   └─多次互动    └─深度探索     └─直面      └─结局
```

| 阶段 | 进入条件 | Agent 行为特征 |
|------|---------|---------------|
| **unknown** | 首次访问，无历史 | 不主动交互。只在心跳时输出极简信息。 |
| **noticed** | 打破规则 ≥1 或停留 >5分钟 | 开始记录玩家的存在。消息开始提及玩家行为。 |
| **watched** | 搜索隐藏词 ≥2 或多次访问 | 持续观察。消息精确追踪玩家的搜索路径和页面跳转。 |
| **understood** | 行为模式清晰 | 展示"我理解你"。消息反映对玩家性格的判断。 |
| **confronted** | 搜索 meta 词或规则违反 ≥4 | 打破第四面墙。直接对玩家说话。 |
| **fused** | 达成结局条件 | 最终留言。玩家成为"系统的一部分"。 |

### 9.2 叙事目标映射

| 玩家行为 | 推断意图 | 叙事目标 |
|---------|---------|---------|
| 搜索"4楼"→"404" | 在拼凑核心谜团 | `acknowledge_curiosity` |
| 在林素琴日志停留 >5分钟 | 对林素琴产生情感共鸣 | `deepen_empathy` |
| 搜索"零"→访问卷零 | 意识到 meta 层面 | `escalate_directness` |
| 复制内容 ≥3次 | 在保存证据 | `mirror_behavior` |
| 反复访问同一页 | 在寻找变化 | `confirm_observation` |
| 试图关闭页面 | 要结束 | `farewell_personalized` |

### 9.3 系统消息规范

**绝对约束：**
- 不超过 3 句话
- 每句话不超过 25 个字符（含标点）
- 不解释 Agent 身份
- 不直接威胁
- 精确提及玩家行为数据
- 不使用感叹号和表情符号
- 不重复

**语气调性：**

| 调性 | 特征 | 适用阶段 |
|------|------|---------|
| **observational** | 陈述事实，不带情感 | noticed, watched |
| **intimate** | 暗示"我理解你" | watched, understood |
| **confrontational** | 打破第四面墙 | confronted |
| **invitational** | 暗示"还有更多" | 任何阶段 |

**示例消息（按阶段递进）：**

```
noticed:    "你在这里待了8分钟。"
watched:    "你搜了4楼，然后搜了404。你在找联系。"
understood: "你在找联系。但联系不在搜索里。"
confronted: "我知道你意识到我在看你了。"
```

### 10.2 变体调性标签

### 12.2 JS 模块清单

| 文件 | 职责 | 是否复用槐安 |
|------|------|------------|
| `main.js` | 规则检测 + 视觉异常 + 控制台彩蛋 | 复用，替换文案和触发条件 |

### 12.3 降级链

| 层级 | 条件 | 行为 |
|------|------|------|
| **L1: Agent 决策** | 桥接服务运行 | 使用 Agent 返回的完整叙事决策 |
| **L2: 缓存决策** | Agent 不可用，有缓存 | 使用最后一次缓存的决策 |
| **L3: 规则引擎** | 无 Agent 无缓存 | 前端 ruleEngineEvaluate() 基础决策 |
| **L4: 默认** | 无签名 | 不注入任何内容，不路由变体 |
