# 白鹿疗养院 · 叙事文本撰写设计文档

> 版本：v1.0
> 日期：2026-06-05

---

## 一、内容源架构

所有叙事内容以 Markdown 形式存放在 `content/` 目录，构建时通过 `scripts/build-pages.ts` 转换为 HTML。

```
content/
├── meta/              # 框架页面
├── volumes/           # 正编病历（24卷）
├── supplements/       # 补遗/个人档案
├── peripherals/       # 机构外围档案
└── variants/          # 变体页
```

---

## 二、Frontmatter 规范

每个 `.md` 文件必须包含 YAML frontmatter，定义页面元数据。

### 通用字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | 是 | 页面唯一标识，对应输出文件名 |
| `title` | string | 是 | 页面标题，注入 `<title>` 和 `<h2>` |
| `category` | string | 是 | 分类：meta / volume / supplement / peripheral / variant |
| `template` | string | 是 | 使用的 HTML 模板：base / volume / supplement / peripheral / meta |
| `page_num` | string | 是 | 页脚显示：`06/24`、`25/24`、`--/24` 等 |

### 分类特有字段

#### volume（正编病历）

| 字段 | 说明 |
|------|------|
| `volume_no` | 卷号数字，如 `1` |
| `subtitle` | 副标题，如 `入院登记志` |
| `year` | 时间范围，如 `1998–2003` |
| `department` | 所属科室，如 `内科`、`精神科` |

#### supplement（补遗）

| 字段 | 说明 |
|------|------|
| `author` | 作者，如 `林素琴` |
| `author_meta` | 作者生平，如 `1960–2015 · 护士长` |
| `years` | 时间跨度 |

#### peripheral（外围档案）

| 字段 | 说明 |
|------|------|
| `source` | 来源机构，如 `市第三人民医院药房` |
| `archive_id` | 档案编号 |

### Frontmatter 示例

```yaml
---
id: volume-04
title: 卷四 · 精神科评估志
category: volume
template: volume
volume_no: 4
subtitle: 精神科评估志
year: 2005–2010
department: 精神科
page_num: 10/24
---
```

```yaml
---
id: supplement-lin
title: 林素琴护士长的值班日志
category: supplement
template: supplement
author: 林素琴
author_meta: 1960–2015 · 护士长
years: 1998–2015
page_num: 25/24
---
```

## 三、Markdown 正文规范

### 3.1 通用规则

- 使用标准 Markdown 语法
- 段落之间空一行
- 使用 `##` 作为一级标题，`###` 作为二级标题
- 引用块 `>` 用于档案备注、编者按
- 列表用于条目化内容

### 3.2 内容模块标记

变体页中需要被 Agent 提取的内容模块，用 HTML 注释标记：

```markdown
<!-- module:fourth-floor-alert -->
<div class="awakened-insert">
<span class="system-tag">[系统插入 · 访问者定向]</span>
<p>读者若查阅 4 楼病历，请确认您所在建筑的楼层。</p>
<p>本院建筑共 3 层。</p>
</div>
<!-- /module:fourth-floor-alert -->
```

构建器会提取 `<!-- module:{id} -->` 到 `<!-- /module:{id} -->` 之间的内容，注册到内容模块索引中。

### 3.3 手写体段落

林素琴的日志中使用楷体手写体，用 `class="handwritten"` 标记：

```markdown
<p class="handwritten">今天4楼的灯又亮了。我知道这栋楼只有3层。</p>
```

### 3.4 录音/访谈标记

```markdown
<p class="transcript-marker">【2003年4月7日 · 值班室】</p>

<p><strong>林素琴：</strong>院长，4楼又有人按铃了。</p>
<p><strong>院长：</strong>林护士，这栋楼只有3层。</p>
```

## 四、正编病历写作指南

### 4.1 风格

- **表面**：客观、冷静、医学文档体
- **异常**：在客观记录中嵌入不合常理的细节

### 4.2 异常埋入技巧

| 技巧 | 示例 |
|------|------|
| 数据矛盾 | 体温单中患者出现 36.0°C 整持续 7 天 |
| 跨卷引用 | 卷四提到"患者反复提及4楼"，卷八记载"建筑仅3层" |
| 目击者差异 | 同一事件，不同记录者的描述存在微妙矛盾 |
| 时间异常 | 某记录的时间戳早于事件发生时间 |
| 多出一份 | 餐食、药物、病历份数比人数多一 |

### 4.3 每卷结构

```markdown
## 一、概述
[科室背景、时间范围]

## 二、典型案例
[3-5个病例，前2个正常，后几个逐渐异常]

## 三、年度统计
[数据表格，隐藏异常模式]

## 四、特殊情况记录
[更直接的异常描述]

---
【本卷终】
```

## 五、补遗写作指南

### 5.1 林素琴日志

- **时间跨度**：1998（入职）→ 2015（死亡）
- **风格演变**：
  - 早期：工作记录，字迹工整
  - 中期：开始出现困惑和疑问
  - 晚期：恐惧、混乱、手写体潦草
- **核心数字**：**7**（值夜年数、日志册数、异常患者共同点）
- **关键道具**：第7本日志（未完整编目）

### 5.2 日志条目格式

```markdown
### 【日期】

[正文]

> 有时加入边注或事后补充
```

## 六、变体页创作规范

### 6.1 变体标签定义

| 标签 | 调性 | 内容特征 |
|------|------|---------|
| `awakened` | 直接打破第四面墙 | `.awakened-insert` 区块，SYSTEM 标签，第二人称 |
| `empath` | 情感向 | 更多林素琴内心描写，温柔语气 |
| `curious` | 引导向 | 更多线索指向，"详见卷X" |
| `deep` | 深层真相 | 所有隐藏内容展示，无保留 |

### 6.2 变体页命名

```
{id}-{variant}.md

例：
- supplement-lin-awakened.md
- volume-04-awakened.md
- volume-00-empath.md
```

### 6.3 变体页与基础页的关系

- 变体页**不重复**基础页的全部内容
- 变体页在基础页内容上**追加或替换**特定区块
- 使用 `<!-- module:{id} -->` 标记可被 Agent 提取的区块

## 七、外围档案写作指南

### 7.1 功能定位

外围档案提供**外部视角**，与正编病历形成交叉验证或矛盾。

### 7.2 来源机构清单

| 档案 | 来源 | 与正编的交叉点 |
|------|------|---------------|
| 药房发药记录 | 市第三人民医院药房 | 某药物发放量 > 患者人数 |
| 监控备份 | 安保公司 | 画面中出现"多出来的人" |
| 保洁记录 | 保洁公司 | 4楼的清洁工单 |
| 餐饮配送单 | 餐饮供应商 | 餐食份数 = 患者人数 + 1 |
| 急救出车记录 | 急救中心 | 疗养院的"非紧急"呼叫 |
| 殡仪馆交接记录 | 殡仪馆 | 遗体数量 ≠ 死亡记录 |
| 停电记录 | 供电所 | 特定时间的断电与恢复 |

### 7.3 格式

外围档案使用**表格体**或**清单体**，模拟真实机构文档：

```markdown
## 2003年4月药品发放记录

| 日期 | 药品名称 | 规格 | 数量 | 领用人 | 备注 |
|------|---------|------|------|--------|------|
| 04-07 | 艾司唑仑片 | 1mg×20 | 31盒 | 精神科 | 当日住院患者30人 |
| 04-14 | 艾司唑仑片 | 1mg×20 | 31盒 | 精神科 | 当日住院患者30人 |
```

## 八、规则怪谈写作（notice.md）

### 8.1 结构

```markdown
---
id: notice
title: 访客须知
category: meta
template: meta
page_num: 02/24
---

## 白鹿疗养院病历数字化项目 · 访客须知

1. [规则1]
2. [规则2]
3. [规则3]
4. [规则4]
5. [规则5]
6. 本须知共 5 条。
```

### 8.2 6 条规则

| 规则 | 内容 | 违反行为 |
|------|------|---------|
| 1 | 病历编号自 001 起，无 000 号 | 访问卷零或搜索"零" |
| 2 | 所有文字均为黑色，红字是 OCR 错误 | 注视红字 >3 秒 / 复制红字 |
| 3 | 搜索框自动出现内容不要回车 | 搜索注入的关键词 |
| 4 | 单份病历阅读不超过 10 分钟 | 单页停留 >10 分钟 |
| 5 | 本院未设留言功能 | 5 分钟内访问 >5 页 |
| 6 | **本须知共 5 条** | 仔细阅读并发现数量矛盾 |

## 九、构建输出

### 9.1 模板变量

构建器将 frontmatter 字段注入 HTML 模板：

| 模板变量 | 来源 |
|----------|------|
| `{{title}}` | frontmatter.title |
| `{{page_num}}` | frontmatter.page_num |
| `{{content}}` | Markdown 正文转 HTML |
| `{{id}}` | frontmatter.id |
| `{{author}}` | frontmatter.author |
| `{{year}}` | frontmatter.year |

### 9.2 模板选择

| template | 用途 |
|----------|------|
| `base` | 通用页面 |
| `meta` | 首页、须知、关于 |
| `volume` | 正编病历 |
| `supplement` | 补遗/个人档案 |
| `peripheral` | 外围档案 |

## 十、内容校验清单

每篇内容提交前检查：

- [ ] frontmatter 包含所有必填字段
- [ ] `id` 全局唯一
- [ ] `page_num` 符合设定（正编 01-24，补遗可 >24，卷零为 00）
- [ ] 跨卷引用准确（如"详见卷八"实际存在）
- [ ] 异常线索可被发现（不简单直白，也不过于隐蔽）
- [ ] 变体页包含至少一个 `<!-- module:{id} -->` 标记
- [ ] 林素琴日志的时间线一致
- [ ] 外围档案数据与正编不矛盾（或矛盾是故意的且有解释）
